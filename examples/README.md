# Examples

## Purpose

This directory contains representative example files generated by the DTrace workflow.

The examples demonstrate the intermediate data formats used throughout the engineering traceability pipeline without exposing an entire production design database.

These files are intended for understanding the processing flow and data structures used by DTrace.

---

# Directory Contents

```text
examples/
├── sample_schematic_checkpoint.csv
├── sample_schematic_checkpoint.json
└── sample_adexl_results.json
```

---

# Example Files

## sample_schematic_checkpoint.csv

Raw schematic checkpoint exported by the SKILL extraction layer.

This file contains information collected directly from the active schematic, including:

* instance names
* library names
* cell names
* view names
* selected device parameters

The CSV serves as the interchange format between the Cadence SKILL environment and the Python processing engine.

---

## sample_schematic_checkpoint.json

Structured representation of the schematic checkpoint.

Generated by:

```text
schematic_tool.py
```

The JSON representation preserves the complete schematic snapshot in a format that is easier to inspect, compare, and process programmatically.

---

## sample_adexl_results.json

Structured representation of exported ADE XL simulation results.

Generated by:

```text
parse_adexl.py
```

The parsed data includes:

* measurement names
* nominal values
* specification limits
* pass/fail status
* minimum and maximum values
* process corners
* temperatures
* corner-specific simulation values

This file serves as the input for the simulation comparison engine.

---

# Relationship Between Files

The examples illustrate the progression from raw exported data to structured engineering information.

```text
SKILL Extraction
        │
        ▼
sample_schematic_checkpoint.csv
        │
        ▼
schematic_tool.py
        │
        ▼
sample_schematic_checkpoint.json
```

Simulation results follow a similar path.

```text
ADE XL Export
        │
        ▼
CSV
        │
        ▼
parse_adexl.py
        │
        ▼
sample_adexl_results.json
```

---

# Notes

The example files included in this directory have been prepared for demonstration purposes.

They preserve the structure of actual DTrace outputs while avoiding unnecessary project-specific information.

Their purpose is to help users understand the processing pipeline rather than reproduce a complete design project.

---

# Summary

The files in this directory demonstrate how DTrace transforms exported engineering data into structured representations that can be compared across checkpoints and incorporated into engineering reports.
